# Universal2MD P0 阶段交付走查

根据批准的 PRD，我已完成 Universal2MD 本地转换工具 P0 阶段核心功能的开发。

## 架构选型与落实
1. **容器层**：基于 `Tauri 2 + React + TypeScript` 构建，具备轻量化的桌面端基因。
2. **核心解析层**：如 PRD 建议，我们在底层集成了 `pandoc` 的子进程调用。由于 `pandoc` 原生提供了惊艳的 `--extract-media` 功能，完美解决了文档解析中的**图文分离**这一硬约束，图片会自动抽离并保存在 `assets/` 目录中。
3. **UI 层**：采用双栏布局设计（左侧任务队列，右侧渲染与操作）。设计语言采用了现代的玻璃态高亮色系及清晰的流转状态（Pending / Processing / Success / Error）。

## 核心功能清单

### ✅ 1. 多文件队列与选择
- **本地安全**：使用 Tauri 的原生 `plugin-dialog` 直接调用系统文件选择器，后续也可无缝支持 Drag & Drop。
- **状态追踪**：左侧列表实时反映文档的状态（灰色待处理、蓝色旋转解析中、绿色成功）。

### ✅ 2. 一键批量转换
- 点击底部“开始转换”后，应用将遍历队列中的任务并通过 IPC 发送至 Rust 后端处理。
- 后端采用高效的 `std::process::Command`，为每个文档生成对应的输出文件夹及其内部的 Markdown 文件和静态资源。

### ✅ 3. 实时富文本预览与提取
- 转换完成后，右侧立刻通过 `react-markdown` 呈现结构清晰的解析结果。
- 提供了“复制全部”系统快捷按钮，方便复制结果去其他应用中使用。

## 验收剧本执行指南

> [!TIP]
> 确保你的系统已全局安装 `pandoc` 以便底层解析引擎顺利运行（`pandoc --version` 测试）。

**启动方法**：
```bash
cd "/Users/meizu/Documents/00|我的文档/30_Projects (项目执行区)/to-md/universal-to-md"
npm run tauri dev
```

1. **测试 DOCX 提取**：在应用界面点击“添加文件”，选择一个图文混排的 DOCX。
2. **启动引擎**：点击左下角“开始转换”，观察状态从 Loading 到 Success。
3. **查验隔离结果**：转换成功后，去源文件所在目录，你会发现多了一个与文件同名的文件夹，点进去就是干净的 `.md` 以及 `assets` 目录下的图片。
4. **实时预览**：点击左侧成功的记录，右侧即可预览转换后的 Markdown，并一键复制到剪贴板。

## 后续开发计划 (P1)

接下来我们可以进入 P1 阶段：
- **PDF 与小众格式增强**：引入 OCR 或特定 PDF 库，优化目前 Pandoc 对复杂 PDF 排版的识别缺陷。
- **拖放 API 集成**：将现有的“点击添加”无缝升级为区域拖放响应。
- **并行线程池**：目前的批量转换是依次执行的，之后可在 Rust 端引入 `tokio` 实现极速多线程并发处理。
